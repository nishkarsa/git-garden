Essential Git Commands

Master these 6 fundamental commands to build your Git foundation

üèÅ

git init

git init

Initialize a new Git repository in your current directory. This creates a hidden .git folder that tracks all your project changes.

üìä

git status

git status

Check the status of your repository. See which files are modified, staged, or untracked. Your go-to command for understanding your repo's current state.

‚ûï

git add

git add filename.txt
git add .

Stage changes for commit. Use 'git add .' to stage all changes, or specify individual files. Think of it as preparing your changes for the next commit.

üíæ

git commit

git commit -m "Your message here"

Save your staged changes to the repository with a descriptive message. This creates a snapshot of your project at this point in time.

üìú

git log

git log
git log --oneline

View the commit history of your repository. See who made changes, when, and what they did. Use --oneline for a compact view.

git clone

git clone <repository-url>

Copy a repository from GitHub or other remote sources to your local machine. This downloads the entire project history and files.
