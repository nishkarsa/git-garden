Intermediate Git Commands

Expand your Git knowledge with these powerful commands

üåø

git branch

git branch <branch-name>
git branch

Create a new branch or list existing branches. Branches are independent lines of development.

üîÑ

git checkout

git checkout <branch-name>
git checkout -b <new-branch>

Switch to a different branch or create and switch to a new one. Also used to discard changes in a file.

ü§ù

git merge

git merge <branch-name>

Integrate changes from one branch into the current branch. This combines development histories.

üì°

git remote

git remote add origin <url>
git remote -v

Manage connections to remote repositories. 'add origin' links your local repo to a remote one.

‚¨ÜÔ∏è

git push

git push origin <branch-name>

Upload local branch commits to a remote repository. Share your changes with the team.

‚¨áÔ∏è

git fetch

git fetch origin

Download objects and refs from another repository. It fetches changes without merging them.

üì•

git pull

git pull origin <branch-name>

Fetch from and integrate with another repository or a local branch. It's a 'fetch' followed by a 'merge'.

üì¶

git clone

git clone <repository-url>

Create a copy of an existing Git repository. This downloads the entire project history.

üîç

git diff

git diff
git diff --staged

Show changes between commits, commit and working tree, etc. '--staged' shows changes ready for commit.

üóëÔ∏è

git branch -d

git branch -d <branch-name>

Delete a local branch, but only if it has been fully merged into its upstream branch.

üî•

git branch -D

git branch -D <branch-name>

Force delete a local branch, even if it has unmerged changes. Use with caution!

‚è™

git reset

git reset HEAD <file>
git reset --hard <commit>

Unstage changes from the staging area, or revert to a previous commit. Powerful and potentially destructive.

‚úèÔ∏è

git commit --amend

git commit --amend -m "New message"

Change the last commit. This is useful for fixing typos or adding forgotten files to the previous commit.

‚Ü©Ô∏è

git checkout --

git checkout -- <file>

Discard changes in the working directory for a specific file. Reverts the file to its last committed state.

üè∑Ô∏è

git tag

git tag <tag-name>
git tag -a <tag-name> -m "Message"

Create a tag to mark specific points in history as important (e.g., release versions).

üì§

git push origin <branch>

git push origin feature-branch

Push a specific local branch to the remote repository.

üîó

git branch --set-upstream-to

Set up the tracking relationship between a local branch and a remote branch.
